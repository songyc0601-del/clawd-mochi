# AP + STA 双模式 WiFi 设计

## 目标

让 Clawd Mochi 在不影响电脑正常联网的情况下提供网页控制和 OTA。
设备首次配置时允许手机连接 `ClaWD-Mochi` 热点填写现有 WiFi 信息；
配置成功后，电脑和设备通过同一个局域网通信。

## 范围

- 保留现有热点：
  - SSID：`ClaWD-Mochi`
  - 密码：`clawd1234`
  - 地址：`http://192.168.4.1`
- 新增 STA 模式，设备可加入家庭或办公室的 2.4 GHz WiFi。
- 新增网络设置页，用于扫描、保存、连接和清除 WiFi 配置。
- 屏幕在启动阶段显示局域网连接结果和局域网 IP。
- `/state` 增加网络模式、连接状态和 IP 字段。
- 更新中文使用文档。
- 保持现有网页控制、OTA、USB 串口及 Codex 状态推送兼容。

不新增云服务、用户账号、远程公网访问或企业级认证支持。

## 用户流程

### 首次配置

1. 用户用手机连接 `ClaWD-Mochi`。
2. 打开 `http://192.168.4.1`，进入“网络设置”。
3. 页面扫描并列出附近的 WiFi。
4. 用户选择 WiFi、填写密码并保存。
5. 设备尝试连接，同时继续保留自身热点。
6. 连接成功后，页面和屏幕显示设备的局域网 IP。

### 日常使用

1. 设备启动后读取已保存的 WiFi 配置并自动连接。
2. 电脑继续连接原有 WiFi，通过设备局域网 IP 打开控制页或 OTA 页。
3. `ClaWD-Mochi` 热点保持可用，供故障恢复和重新配置。

### 连接失败

- 设备热点保持可用。
- 网络设置页显示连接失败或未连接状态。
- 用户可重新填写配置，或清除已保存配置。
- WiFi 故障不影响 USB 串口控制。

## 固件设计

### 网络模式

启动时使用 `WIFI_AP_STA`：

- 先启动固定地址为 `192.168.4.1` 的设备热点。
- 从 ESP32 `Preferences` 中读取已保存的 SSID 和密码。
- 存在配置时尝试 STA 连接；不存在配置时只保持热点。
- STA 连接过程不得阻塞主循环过久，网页服务和串口处理应继续可用。

### 配置存储

使用 ESP32 `Preferences` 保存两个字符串：

- `ssid`
- `password`

清除操作删除这两个值并断开 STA，设备热点不受影响。

### HTTP 接口

新增页面：

- `GET /network`：显示网络配置页面。

新增接口：

- `GET /wifi/scan`：返回附近 WiFi 列表。
- `POST /wifi/connect`：保存 SSID 和密码，并开始连接。
- `POST /wifi/clear`：清除配置并断开 STA。

接口仅供本地网络使用。第一版不增加登录认证。

`GET /state` 保留现有字段，并增加：

- `wifiMode`：`AP` 或 `AP_STA`
- `wifiConnected`：布尔值
- `wifiSsid`：已连接的 SSID，未连接时为空
- `wifiIp`：局域网 IP，未连接时为空
- `apIp`：固定为 `192.168.4.1`

### 页面整合

现有精简控制页的“设备管理”区域增加一个“网络设置”入口。
网络设置页只展示完成配置所需内容：

- 当前连接状态和局域网 IP
- WiFi 扫描结果
- 密码输入框
- 保存并连接按钮
- 清除配置按钮

页面使用简短中文，不增加手动 Codex 状态按钮。

### 屏幕反馈

启动时短暂显示：

- 热点地址 `192.168.4.1`
- STA 已连接时显示局域网 IP
- 未配置或连接失败时提示使用热点配置

之后进入现有 Codex 待机页。运行期间网络变化不强制打断当前状态页。

## Codex 对接

现有 `tools/codex-stage.ps1` 继续支持 `-DeviceUrl`，用户可以传入设备局域网地址。
本轮不实现自动发现设备，避免引入 mDNS、广播发现或额外依赖。
HTTP 不可用时仍自动回退到 USB 串口。

## 安全与约束

- ESP32-C3 只支持 2.4 GHz WiFi，文档必须明确说明。
- WiFi 密码保存在设备本地 NVS，不通过 `/state` 或扫描接口返回。
- 设备热点始终保留，因此同一局域网内可访问设备网页；第一版不提供身份认证。
- SSID 和密码提交使用 HTTP，适用于可信本地网络，不适用于公共网络。

## 验收标准

- 无 WiFi 配置时，设备热点和原有网页功能可用。
- 手机可通过网络设置页扫描、保存并连接一个 2.4 GHz WiFi。
- 连接成功后，屏幕和 `/state` 显示局域网 IP。
- 电脑保持原有 WiFi 连接时，可通过局域网 IP 使用控制页和 OTA。
- 清除配置后 STA 断开，设备热点仍可访问。
- 主程序与 `dist` 版本一致并编译通过。
- 现有网页契约测试、USB 命令和 Codex 状态推送不受破坏。
