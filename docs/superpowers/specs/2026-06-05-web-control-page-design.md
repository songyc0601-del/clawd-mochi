# Clawd Mochi Web 控制页面设计

日期：2026-06-05

## 目标

把现有手机 Web 页面整理成简洁、容易理解的 Clawd Mochi 设备控制台。
页面面向第一次使用设备的用户，重点是快速查看状态和完成常用操作。

## 页面定位

- 页面品牌是 `Clawd Mochi`，不是 Codex。
- Codex 是“工作状态”模块当前已接入的状态来源。
- 工作状态只读，由 Codex 等 App 自动推送，网页不提供手动状态设置。
- 后续可以接入 Claude Code 等 App，但本轮不扩展多 App 协议。

## 首页结构

首页保持单页、手机优先，按以下顺序展示：

1. 顶栏
   - `Clawd Mochi`
   - 设备在线状态
2. 工作状态
   - 当前来源：`Codex`
   - 当前阶段：`IDLE / PLAN / CODE / TEST / DONE / BLOCK`
   - 阶段进度条
3. 屏幕显示
   - 正常
   - 专注
   - 开心
4. 设备
   - 屏幕背光
   - OTA 升级

## 文案与交互

- 页面减少文字内容，只保留必要标签。
- 不显示英文按钮副标题和长说明段落。
- 工作状态区域不出现任何手动设置按钮。
- 点击屏幕显示或背光后立即调用现有接口，并刷新当前状态。
- 操作成功使用短暂中文提示。
- 只有连接失败时显示错误说明。
- OTA 页面统一为简洁中文界面，但不改变上传流程。

## 技术边界

- 复用现有 `/state`、`/cmd`、`/expr`、`/backlight` 和 `/ota` 接口。
- 保留 `/progress`、USB 串口与 PowerShell 推送能力，供 App 自动对接使用。
- 不修改固件协议，不添加 App 来源字段。
- 不添加外部字体、图片、框架或网络资源。
- 页面继续内嵌在 Arduino 固件中。

## 验收标准

- 首页不包含手动 Codex 状态按钮。
- 首页仅展示 Clawd Mochi、在线状态、工作状态、三个屏幕显示、背光和 OTA。
- 手机页面布局清晰，文字不溢出，按钮可直接操作。
- 工作状态能从 `/state` 自动刷新。
- 页面操作继续正确调用现有固件接口。
- OTA 上传流程不受影响。
- 主程序与 `dist` 程序一致并编译通过。
